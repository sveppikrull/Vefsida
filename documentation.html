Project 1 - Documentation

Project description

In this project I will be making a website using HTML. The main purpose of this website will be to host the projects that I will be doing for the course VÉL403G at the University of Iceland. Further more I want to use this opportunity to make a website that I can use in the future, as a portfolio that can help me when applying to a school or for a job. Therefor I will be trying my best to make the website look as professional as I can with my limited expirience using HTML. I did the coding in an application called Brackets.

Making the website

Since I have no experience using HTML I decided to use a template taken from the website https://html5up.net/ to make life easier. At first I chose the template Dimensions as I found the design stunning and the user interface was simple. However, after having spent some time trying to customize the template I discovered that it was maybe too simple, it lacked templates for different types of pages. Therefor I decided to switch to the Helios template as it had multiple templates for different types of pages and the front page user interface was more extensive (had a drop down feature and more).

Most of the time was spent trying out different things and seeing what bits of code influenced which part of the site. Once I felt that I knew how things worked I started trimming down the template. It had a lot of things going on at once. My goal was to make the website look professional and simple. So when I felt I had removed all the unnecessary parts I started to perfect the home page. Once I had finished the home page I started working on the other pages. I decided on having the subpages: About Me, University of Iceland, Personal Work and Resume.

Subpages

On the About Me page I wrote a few paragraphs about myself and my interest and uploaded a picture of me. I spent quite a lot of time on trying to get the page to look good. I managed to get a look that I am somewhat happy with but I will probably change it in the future when I've gotten more comfortable HTML.

For the University of Iceland tab I used a drop down menu where I will link various courses and the big projects I have done. In the dropdown menu there is a link to this course and in particular this project. Furthermore have I linked a few big projects I have done during my studies.

The Personal Work page is rather empty for the time being. Since I haven't done any big projects on my own time I don't have a lot to put on there. So in the meantime I have uploaded a few pictures I have taken over the years. I mainly look at this page as something that I can use in the future.

The Resume tab is just a simple link that pulls up my resume.

Uploading the page to GitHub

I have no experience with GitHub so I followed the instructions laid out on Canvas and got help from my friend Jón Lárus who had already gone through the same process.

1. First you download and install Homebrew by following the instructions on https://brew.sh/.

2. Then you go to https://github.com// and make an account and a repository.

2. Once you have made a repository, go to the terminal and put the directory to the folder where all the HTML files are stored. Then paste the code that the GitHub site gives you into the terminal.

3. Once that is done the file readme.md should be in the repository. Then to add the other files to the repository, you put the following commands in the terminal:
  git add .
  git status
  git commit -m "assisted"
  git push
And always push enter between commands. Now all the relevant files should be in the repository.

4. Then go to your GitHub repository. Under settings change your GitHub page source from none to main. Then after a few minutes the site should be up and running.

5. To update the files in your repository. Go to the terminal and put the directory to the folder where files are stored. Then use the following commands:
  git init
  git add .
  git commit -m "assisted"
  git push
And after a few minutes the files should have been updated in the repository and the site will be updated.

Conclusion

I quite enjoyed this project. It was a fun challenge to make this website and I look at this as an asset that I can use in the future. I will probably be updating the site during this semester and try to make it look as good as I can.
